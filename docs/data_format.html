Data Format

If you use JSON schemas, see https://raw.githubusercontent.com/MRT-Map/gatelogue/dist/schema.json.

If you use Typescript, see https://github.com/MRT-Map/gatelogue/blob/main/gatelogue-client/src/stores/schema.ts for Typescript types.

Specification

  • All links below reference their entries in the full reference page. These classes are here for convenience’s sake.

  • Do not hardcode any UUIDs in your project! They change with every update. If you need to reference a specific object, find it by its code/name/something unique to the object.

  • Check the version field in the Context object for the data format version. We will try our best to maintain backwards-compatibility, but we cannot guarantee.

  • The JSON file at data.json is of base type Context.

  • If you are using data_no_sources.json, all instances of Sourced[T] or Sourced.Ser[T] below are replaced with just the encapsulated type T.

    • If you are using the Typescript types referenced above, the type for the JSON of the no-source version is GatelogueData<false> instead of simply GatelogueData or GatelogueData<true>.

  • Most objects here have a .Ser prefix because they are the serialised versions of their original classes. However in your own projects a Ser suffix is unnecessary.

  • Context objects can be called Data in your project.

  • uuid.UUID serialises to a string, eg. 0b0e1f74-3683-4ff2-aa51-a13587950c56

  • tuple and set serialise to a list.

  • None serialises to null.

The current data format version is

v1
class Context.Ser(*, air: ~gatelogue_aggregator.types.node.air.AirContext.Ser, rail: ~gatelogue_aggregator.types.node.rail.RailContext.Ser, sea: ~gatelogue_aggregator.types.node.sea.SeaContext.Ser, bus: ~gatelogue_aggregator.types.node.sea.SeaContext.Ser, timestamp: str = <factory>, version: int = 1)
air: Ser

An AirContext object

bus: Ser

A BusContext object

rail: Ser

A RailContext object

sea: Ser

A SeaContext object

timestamp: str

Time that the aggregation of the data was done

version: int

Version number of the database format

Air

class AirContext.Ser(*, flight: dict[UUID, Ser], airport: dict[UUID, Ser], gate: dict[UUID, Ser], airline: dict[UUID, Ser])
airline: dict[uuid.UUID, AirAirline.Ser]
airport: dict[uuid.UUID, AirAirport.Ser]
flight: dict[uuid.UUID, AirFlight.Ser]
gate: dict[uuid.UUID, AirGate.Ser]
class AirFlight.Ser(*, codes: set[str], gates: list[Ser[UUID]], airline: Ser[UUID])
airline: Sourced.Ser[uuid.UUID]

ID of the AirAirline the flight is operated by

codes: set[str]

Unique flight code(s). 2-letter airline prefix not included

gates: list[Sourced.Ser[uuid.UUID]]

List of IDs of AirGate s that the flight goes to. Should be of length 2 in most cases

class AirAirport.Ser(*, coordinates: Ser[tuple[int, int]] | None, world: Ser[Literal['New', 'Old']] | None, proximity: dict[str, dict[UUID, Ser[Proximity]]], code: str, name: Ser[str] | None, link: Ser[str] | None, gates: list[Ser[UUID]])
code: str

Unique 3 (sometimes 4)-letter code

coordinates: Sourced.Ser[tuple[int, int]] | None

Coordinates of the object

gates: list[Sourced.Ser[uuid.UUID]]

List of IDs of AirGate s

link: Sourced.Ser[str] | None

Link to the MRT Wiki page for the airport

name: Sourced.Ser[str] | None

Name of the airport

proximity: dict[str, dict[uuid.UUID, Sourced.Ser[Proximity]]]

References all objects that are near (within walking distance of) this object. It is represented as a mapping of object types to a inner mapping of object IDs to proximity data (Proximity). For example, {"airport": {1234: <proximity>}} means that there is an airport with ID 1234 near this object, and <proximity> is a Proximity object.

world: Sourced.Ser[Literal['New', 'Old']] | None

Whether the object is in the New or Old world

class AirGate.Ser(*, code: str | None, flights: list[Ser[UUID]], airport: Ser[UUID], airline: Ser[UUID] | None, size: Ser[str] | None)
airline: Sourced.Ser[uuid.UUID] | None

ID of the AirAirline that owns the gate

airport: Sourced.Ser[uuid.UUID]

ID of the AirAirport

code: str | None

Unique gate code. If None, all flights under this gate do not have gate information at this airport

flights: list[Sourced.Ser[uuid.UUID]]

List of IDs of AirFlight s that stop at this gate. If code==None, all flights under this gate do not have gate information at this airport

size: Sourced.Ser[str] | None

Abbreviated size of the gate (eg. S, M)

class AirAirline.Ser(*, name: str, flights: list[Ser[UUID]], link: Ser[str] | None)
flights: list[Sourced.Ser[uuid.UUID]]

List of IDs of all AirFlight s the airline operates

link: Sourced.Ser[str] | None

Link to the MRT Wiki page for the airline

name: str

Name of the airline

class RailStation.Ser(*, coordinates: Ser[tuple[int, int]] | None, world: Ser[Literal['New', 'Old']] | None, proximity: dict[str, dict[UUID, Ser[Proximity]]], codes: set[str], company: Ser[UUID], connections: dict[UUID, list[Ser[Ser]]], name: Ser[str] | None)

References all objects that are near (within walking distance of) this object. It is represented as a mapping of object types to a inner mapping of object IDs to proximity data (Proximity). For example, {"airport": {1234: <proximity>}} means that there is an airport with ID 1234 near this object, and <proximity> is a Proximity object.

Miscellaneous

class Sourced.Ser(v: _T, s: set[str])
s: set[str]

List of sources that support the value

v: _T

Actual value

class Connection.Ser(*, line: UUID, direction: Ser | None = None)
direction: Direction.Ser | None

Direction information

line: uuid.UUID

ID of the RailLine/SeaLine/BusLine that the connection is made on

class Direction.Ser(*, forward_towards_code: UUID, forward_direction_label: str | None, backward_direction_label: str | None, one_way: bool)
backward_direction_label: str | None

Describes the direction taken when travelling from the station/stop in forward_towards_code

forward_direction_label: str | None

Describes the direction taken when travelling towards the station/stop in forward_towards_code

forward_towards_code: uuid.UUID

ID of the RailStation/SeaStop/BusStop that the other fields take with respect to. Should be either of the stations/stops of the connection

one_way: bool

Whether the connection is one-way, ie. travel towards the station/stop in forward_towards_code is possible but not the other way

class gatelogue_aggregator.types.connections.Proximity(distance: int)
distance: int

Distance between the two objects in blocks
